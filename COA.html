Features

            
            

                
                
                

                    

                        
Number System Conversion

                        
                    

                    

                        
What it is: A number base (or *radix*) is the number of unique digits used to represent numbers. This calculation converts a number from one base to another.

                        
Why it's Used: This is the most fundamental concept in COA. It explains how humans (Base 10) communicate with computers, which operate in Base 2 (Binary).

                    

                

                
                
                

                    

                        
1's & 2's Complement

                        
                    

                    

                        
What it is: The standard method computers use to represent and handle negative numbers in binary.

                        
Why it's Used: It massively simplifies hardware. Using 2's complement, subtraction (A minus B) becomes simple addition (A plus the 2's complement of B). This means the Arithmetic Logic Unit (ALU) only needs an "adder" circuit.

                    

                

                
                
                

                    

                        
Decimal to BCD

                        
                    

                    

                        
What it is: Binary-Coded Decimal (BCD) encodes each *individual decimal digit* as a 4-bit binary group (a *nibble*).

                        
Why it's Used: It's vital in financial systems, calculators, and digital clocks where decimal rounding must be precise and interfacing with decimal displays is common.

                    

                


                
                

                    

                        
Decimal to Excess-3

                        
                    

                    

                        
What it is: Excess-3 is a non-weighted, "self-complementing" BCD code. You add 3 to each decimal digit *before* converting it to a 4-bit nibble.

                        
Why it's Used: Its "self-complementing" nature. The 9's complement of a decimal digit can be found by simply flipping the bits (1's complement) of its Excess-3 code. This simplified subtraction hardware in older computers.

                    

                


                
                

                    

                        
Grey Code Converter

                        
                    

                    

                        
What it is: Grey Code is a "unit-distance" binary code where only *one single bit* changes between any two consecutive numbers.

                        
Why it's Used: It prevents errors in electro-mechanical sensors (like a robotic arm) and is used in logic minimization (Karnaugh maps).

                    

                


                
                

                    

                        
Binary Arithmetic

                        
                    

                    

                        
What it is: The basic math (add, subtract) that the CPU's Arithmetic Logic Unit (ALU) performs at its lowest level.

                        
Key Concepts: Addition follows simple rules (1+1=0, carry 1). Subtraction is performed by adding the 2's complement, simplifying the hardware design.

                    

                


                
                

                    

                        
BCD Arithmetic

                        
                    

                    

                        
What it is: Addition performed on BCD-coded numbers.

                        
The "Fix": If adding two 4-bit nibbles results in a value greater than 9, the result is an invalid BCD code. To correct this, you must add 6 (0110) to that nibble. This adjustment produces the correct BCD result.

                    

                


                
                

                    

                        
Booth's Algorithm

                        
                    

                    

                        
What it is: Booth's Algorithm is a fast, efficient algorithm for multiplying two *signed* (2's complement) binary numbers.

                        
Why it's Used: It's much faster than simple repeated addition. It can skip over long strings of 1s or 0s, reducing the number of operations.

                    

                


                
                

                    

                        
Restoring Division

                        
                    

                    

                        
What it is: Restoring Division is a basic algorithm for *unsigned* division that mimics "pen-and-paper" long division.

                        
Algorithm: It tries to subtract the divisor (M). If the result is negative, it sets the quotient bit to 0 and *restores* the original value by adding M back. If positive, it sets the bit to 1. It's simple but slow.

                    

                

                
                
                

                    

                        
Non-Restoring Division

                        
                    

                    

                        
What it is: Non-Restoring Division is a faster, more complex algorithm that avoids the time-wasting "restore" step.

                        
Algorithm: If the accumulator is positive, it subtracts M. If it's negative, it *adds* M. It skips the restore cycle by compensating in the *next* cycle, making it more efficient.

                    

                


                
                

                    

                        
IEEE 754 Converter

                        
                    

                    

                        
What it is: IEEE 754 is the universal standard format for "floating-point" (decimal) numbers in computing.

                        
Structure (32-bit): It's composed of a 1-bit Sign, an 8-bit Exponent (with a bias of 127), and a 23-bit Mantissa (the precision digits).
